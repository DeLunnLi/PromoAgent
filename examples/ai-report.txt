Source brief
Title: Source2Launch
Type: repository
Summary: A CLI that reads open-source projects or papers and drafts grounded cross-platform promotion copy.

Target readers
- open source maintainers
- developer tool builders
- researchers who share papers online

Key points
- Starts from repository or paper evidence instead of a blank prompt
- Generates tweets, Zhihu answers, Xiaohongshu carousel notes, WeChat article drafts, and image ideas
- Can preview image-generation requests without exposing provider API keys

Single tweet
I built Source2Launch to turn a repo or paper into grounded promotion copy: tweet, thread, Zhihu answer, Xiaohongshu note, WeChat article, hashtags, and image ideas. The goal is simple: less blank-page writing, fewer unsupported claims.

Thread
1. Most technical posts start from a vague prompt. Source2Launch starts from source evidence: README, package metadata, examples, paper text, or a URL.
2. It asks the model to separate what is supported by evidence from what needs human confirmation.
3. Then it generates publish-ready variants for X/Twitter, Zhihu, Xiaohongshu, WeChat, and social image prompts.
